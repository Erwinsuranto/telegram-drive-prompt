Tahap 27 — File Version History

Jangan membaca seluruh project atau dokumentasi agar proses lebih cepat.

Periksa hanya file yang berhubungan dengan Upload, My Files, Preview, Download, Storage, Activity, Dashboard, Search, dan metadata file.

Jika fitur Version History sudah ada dan berjalan dengan baik, jangan membuat ulang.

Jika belum ada, implementasikan fitur berikut dan integrasikan dengan sistem yang sudah ada.

Fitur yang harus tersedia:

• Version History untuk setiap file.
• Menampilkan daftar seluruh versi file.
• Informasi versi (tanggal, ukuran, uploader).
• Preview setiap versi.
• Download versi tertentu.
• Restore versi lama menjadi versi aktif.
• Hapus versi tertentu.
• Batas jumlah versi dapat dikonfigurasi.
• Label versi terbaru.
• Search pada Version History.
• Integrasi dengan Upload.
• Integrasi dengan Download.
• Integrasi dengan Preview.
• Integrasi dengan Activity History.
• Integrasi dengan Dashboard.
• Mobile friendly.
• Dark mode.
• Animasi halus.
• Tidak membuat duplicate code.
• Gunakan komponen yang sudah ada.
• Jangan membuat file .md.
• Jangan mengubah backend jika tidak benar-benar diperlukan.
• Jangan merusak fitur sebelumnya.

Sebelum selesai lakukan self-test dan pastikan tidak ada error, import rusak, route rusak, JavaScript error maupun konflik dengan fitur sebelumnya.

Tampilkan ringkasan Stage 27 setelah selesai.
