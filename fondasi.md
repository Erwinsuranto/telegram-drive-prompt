# 📌 Aturan Pengembangan

- Kerjakan satu tahap hingga selesai sebelum lanjut ke tahap berikutnya.
- Jangan mengubah struktur tahap sebelumnya tanpa alasan yang jelas.
- Gunakan GitHub sebagai source of truth.
- Semua perubahan harus memiliki commit yang jelas.
- Dokumentasikan setiap perubahan pada CHANGELOG.md.
- Pastikan setiap tahap dapat dijalankan tanpa error sebelum melanjutkan.
